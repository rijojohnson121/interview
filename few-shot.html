Few-Shot Prompting

Supercharging Test Automation with Playwright

💡 What is Few-Shot Prompting?

Instead of asking AI to "figure it out," you provide 2-3 examples of the desired input-output pattern. This dramatically improves consistency and accuracy—critical for generating reliable test scripts.

⚡

Consistency

Same pattern → Same quality output every time

🎭

Context Awareness

AI understands your testing conventions instantly

⏱️

Speed

Reduce prompt engineering iterations by 70%

EXAMPLE 1

General Pattern Recognition

few-shot-prompt.txt
# Task: Convert informal text to professional email

Input: "hey, meeting moved to 3pm"
Output: "Hi team, Please note that our meeting has been rescheduled to 3:00 PM."

Input: "can't make it today, sick"
Output: "I regret to inform you that I am unwell and unable to attend today."

Input: "need the report by friday"
Output: "Could you please submit the report by end of day Friday?"
EXAMPLE 2

Playwright Test Generation

playwright-few-shot.js
// Task: Generate Playwright tests from user stories

Example 1:
Story: "User logs in with valid credentials"
Test: test('user can login', async ({ page }) => {
  await page.goto('/login');
  await page.fill('[data-testid="email"]', 'user@test.com');
  await page.fill('[data-testid="password"]', 'pass123');
  await page.click('button[type="submit"]');
  await expect(page).toHaveURL('/dashboard');
});

Example 2:
Story: "User adds item to cart"
Test: test('add item to cart', async ({ page }) => {
  await page.goto('/products');
  await page.click('[data-testid="product-1"]');
  await page.click('button:has-text("Add to Cart")');
  await expect(page.locator('.cart-count')).toHaveText('1');
});

Now Generate:
Story: "User searches for 'laptop' and applies price filter"
→ AI generates consistent Playwright code following the exact patterns above!

Pro Tip for QA Engineers

Create a prompt library with your team's testing patterns (POM structure, naming conventions, assertion styles). Few-shot prompting ensures AI generates tests that match your codebase standards—reducing review time and technical debt.
